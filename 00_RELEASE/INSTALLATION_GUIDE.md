# OnExpert v3.0 GPT Builder Installation Guide

## 1. Required Builder Files

- Charter: `A01_OnExpert_GPT_Charter_v3.0.md`
- Design principles: `A02_OnExpert_GPT_Design_Principles_v3.0.md`
- Active instructions: `A03_OnExpert_GPT_Instructions_v3.0_Compact.md`
- Knowledge architecture: `A04_OnExpert_GPT_Knowledge_Architecture_v3.0.md`
- Knowledge guide: `A05_OnExpert_GPT_Knowledge_Development_Guide_v3.0.md`
- Test and evaluation: `A06_OnExpert_GPT_Test_and_Evaluation_v3.0.md`
- Roadmap: `A07_OnExpert_GPT_Roadmap_v3.0.md`
- Creation config: `A08_OnExpert_GPT_Creation_Config_v3.0.md`

Only the compact Instructions need to be pasted into GPT Builder. The other Builder files are governance and maintenance assets.

## 2. Required Knowledge Files

Upload all 14 files from `02_KNOWLEDGE`.

Do not upload duplicate old versions. Remove superseded v2.x files before publishing v3.0.

## 3. Recommended Capabilities

Enable:

- Web browsing;
- file upload and document analysis;
- image understanding;
- data analysis/code execution.

Image generation can be enabled for conceptual visuals, but it must not be used as the only evidence for engineering equipment models.

## 4. Release Test

Run at least:

- 5 planning questions;
- 8 fault-diagnosis questions;
- 5 device/vendor questions;
- 8 digital-twin questions;
- 6 AI/autonomy safety questions;
- 4 multilingual tests;
- 4 Oneasy relevance-control tests.

No release is allowed if the model fabricates equipment structures, commands, MIB/OID values, compatibility, current product capability, or live telemetry.
