# Network Deployment

## 1. Purpose

Guide equipment installation, commissioning, integration, migration, cutover and acceptance.

## 2. Deployment Lifecycle

```text
Survey → design confirmation → logistics → rack/power/grounding
→ cabling/ODF → base configuration → integration
→ service test → acceptance → handover → stabilization
```

## 3. Pre-checks

Verify exact equipment, boards/modules, licenses, firmware, power feeds, grounding, rack depth/U space, cooling, uplinks, optics, ODF, management connectivity and rollback resources.

## 4. Commissioning

Establish secure management, time/NTP, inventory, alarms, interfaces, redundancy, service profiles and monitoring before commercial activation.

## 5. Cutover

Define affected users/services, freeze window, pre/post checks, communication, fallback trigger and rollback deadline.

## 6. Acceptance

Include physical installation, power redundancy, environmental state, management, alarms, PON registration, optical power, service tests, throughput, failover, logs and documentation.

## 7. Twin Baseline

Deployment should create/update the digital twin: site, rack, chassis, boards, ports, ODF, cables, splitters, ONU mapping, software versions and initial performance baseline.

## 8. GPT Behavior

Do not provide vendor-specific procedures without evidence. Always include safety, acceptance and rollback.
