# OnExpert GPT Roadmap v3.0

## Strategic Path

```text
Trusted Optical Access Expert
→ Engineering Digital Twin Expert
→ Oneasy Read-only Copilot
→ Simulation-guarded Agent
→ AI-native Autonomous OAM
```

## v3.0 — Complete Knowledge and Instruction Release

- 14 Knowledge files;
- engineering twin methods;
- TwinGraph and temporal reasoning;
- safety and autonomy standards;
- Builder and release tests.

## v3.1 — Verified Device Twin Library

Create evidence-backed templates for priority Huawei, ZTE, FiberHome and small-OLT ecosystems.

## v3.2 — Operational Twin Prototype

Connect telemetry, alarms, temperature, power, capacity, optical metrics and history. Clearly label simulated vs real data.

## v3.3 — Causal Diagnosis

Connect device, ODN, ONU, configuration, service and customer impact through TwinGraph.

## v4.0 — Oneasy Read-only Integration

Introduce AI Gateway, tenant isolation, redaction, read-only tools, audit and object-grounded TwinCopilot.

## v4.5 — SafeAction Sandbox

Add dry run, compatibility validation, impact simulation, approval and rollback verification.

## v5.0 — Controlled Autonomous OAM

Enable selected low-risk closed loops, progressive rollout, outcome verification and safe rollback.

## Target Maturity

| Stage | Twin | Autonomy |
|---|---|---|
| v3.0 | DT2 knowledge capability | AO1 |
| v3.2 | DT3 operational prototype | AO1–AO2 |
| v3.3 | DT4 causal diagnosis | AO2–AO3 |
| v4.5 | DT4 simulation and governance | AO3 |
| v5.0 | DT5 selected domains | AO4; narrow AO5 only |
